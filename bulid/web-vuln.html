Common Web Vulnerabilities

SQL Injection

Allows attackers to manipulate database queries.

SQL injection occurs when user input is directly included in SQL queries without proper validation. Attackers can modify queries to extract, modify, or delete data.

Example: ' OR '1'='1' can be injected to bypass authentication.

XSS (Cross-Site Scripting)

Injects malicious scripts into web pages.

XSS vulnerabilities arise when applications accept user input without validation and output it to users. Attackers can inject scripts that execute in the context of another userâ€™s browser.

Example: Injecting a script that steals cookies.

CSRF (Cross-Site Request Forgery)

Forces users to perform unwanted actions.

CSRF tricks a user into submitting a request to a web application where they are authenticated, potentially causing unwanted actions like fund transfers.

Example: Sending a request to change user email without consent.

Command Injection

Executes arbitrary commands on the server.

Command injection occurs when an application passes unsafe user input to a system shell. Attackers can execute commands with the same privileges as the application.

Example: Injecting ; ls -la to list directory contents.

File Inclusion

Includes files from the server or remote sources.

File inclusion vulnerabilities allow attackers to include files on a server or remote file, which can lead to code execution or unauthorized access to sensitive files.

Example: Using ?page=../../etc/passwd to access sensitive files.

Insecure Direct Object References

Accesses unauthorized data or files.

This vulnerability occurs when an application exposes a reference to an internal object, allowing users to access objects they should not be able to.

Example: Changing a URL parameter to access another user's data.

Security Misconfiguration

Improper configuration leads to vulnerabilities.

Security misconfigurations can occur at any level of an application stack, including cloud storage, databases, and web servers, potentially exposing sensitive data.

Example: Default credentials left unchanged.

Broken Authentication

Exploiting weaknesses in authentication mechanisms.

Broken authentication allows attackers to compromise user accounts, often due to poor session management or weak password policies.

Example: Predictable session identifiers or lack of account lockout mechanisms.

Sensitive Data Exposure

Inadequate protection of sensitive information.

Sensitive data exposure can occur when sensitive information is not properly encrypted or is transmitted over insecure channels.

Example: Sending passwords in plain text over HTTP.

Insufficient Logging & Monitoring

Failure to log and monitor can allow for undetected attacks.

Without proper logging and monitoring, it becomes difficult to detect and respond to attacks. This can prolong the duration of a breach.

Example: Not logging failed login attempts.
